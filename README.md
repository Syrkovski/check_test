# Генератор БСО

Минимальное веб‑приложение для генерации БСО ИП без кассы. Работает целиком в браузере: сохраняет данные в `localStorage`, автоматически нумерует документы по формуле `YYYYMMDD-XXX`, генерирует PDF по фиксированному шаблону и ведёт реестр выданных бланков.

## Как запустить
1. Откройте файл `index.html` в браузере (двойной клик или `python -m http.server` в папке проекта).
2. При необходимости задайте принудительный номер будущего бланка — он применится один раз, затем вернётся авто-нумерация.
3. Заполните форму «Создать БСО»: укажите количество занятий и цену за 1 занятие; поля «Клиент», «Услуга», «Способ оплаты» уже заполнены фиксированными значениями и при желании разблокируются кнопками справа. Дата и время подставляются автоматически на момент генерации.
4. Нажмите «Создать PDF». Запись попадёт в реестр, номер сформируется автоматически (или по заданному значению), файл скачается. Для построения файла нужен доступ к интернету, чтобы подгрузить `jsPDF`.
5. Реквизиты продавца можно скорректировать через кнопку «Изменить реквизиты» — изменения сохраняются в браузере. Кнопка «Сбросить к файлу» возвращает значения из `vendor.json`.
6. Повторные загрузки PDF доступны из таблицы «Реестр БСО». Реестр можно выгрузить в JSON или загрузить из файла через кнопки над таблицей.

## Что реализовано
- **Авто-нумерация:** `YYYYMMDD-XXX`, счётчик ежедневно сбрасывается.
- **Фиксированные поля:** дата (сегодня), время (текущее при генерации), клиент, услуга, количество занятий, цена за занятие, сумма, способ оплаты, реквизиты ИП из файла `vendor.json`. Клиент («Физлицо»), услуга («Образовательные услуги») и способ оплаты («безналичная (перевод по QR-коду на расчётный счёт ИП)») выводятся статично и редактируются по кнопке справа от поля.
- **PDF:** квитанция с секциями «Продавец», «Покупатель», «Основание платежа», «Состав расчёта», «Оплата», «Подтверждение оплаты» по образцу: разделители вида `------`/`======`, формат даты `ДД.ММ.ГГГГ` и строка времени «за ЧЧ:ММ», встроенный шрифт для корректной кириллицы, кнопки «Скачать PDF» в реестре. В составе расчёта выводятся цена за занятие, количество и итоговая сумма.
- **Нумерация:** авто `YYYYMMDD-XXX` со сбросом по дате, плюс форма принудительного задания следующего номера.
- **Реестр:** сохраняется в `localStorage`, отображает номер, дату/время, клиента, услугу, сумму, способ оплаты, даёт повторное скачивание PDF и удаление записей. Доступна выгрузка реестра в JSON и загрузка из файла.
- **Данные продавца:** вынесены в `vendor.json` и подгружаются при инициализации; дополнительно можно изменить их через встроенную форму (с сохранением в браузере) или сбросить к данным файла.

## Ограничения MVP
- Один пользователь, без аутентификации.
- Минималистичный дизайн: белый фон, чёрный текст, без адаптивной красоты.
- Данные хранятся только локально в браузере; при очистке локального хранилища история пропадает.
