# Генератор БСО

Минимальное веб‑приложение для генерации БСО ИП без кассы. Работает целиком в браузере: сохраняет данные в `localStorage`, автоматически нумерует документы по формуле `YYYYMMDD-XXX`, генерирует PDF по фиксированному шаблону и ведёт реестр выданных бланков.

## Как запустить
1. Откройте файл `index.html` в браузере (двойной клик или `python -m http.server` в папке проекта).
2. Заполните форму «Создать БСО»: сумма, время и при необходимости комментарий; поля «Клиент», «Услуга», «Способ оплаты» уже заполнены фиксированными значениями и при желании разблокируются кнопками справа.
3. Нажмите «Создать PDF». Запись попадёт в реестр, номер сформируется автоматически, файл скачается. Для построения файла нужен доступ к интернету, чтобы подгрузить `jsPDF`.
4. Повторные загрузки PDF доступны из таблицы «Реестр БСО».

## Что реализовано
- **Авто-нумерация:** `YYYYMMDD-XXX`, счётчик ежедневно сбрасывается.
- **Фиксированные поля:** дата (сегодня), время (ручной ввод), клиент, услуга, сумма, способ оплаты, комментарий, реквизиты ИП из файла `vendor.json`. Клиент («Физлицо»), услуга («Образовательные услуги») и способ оплаты («безналичная (перевод по QR-коду на расчётный счёт ИП)») выводятся статично и редактируются по кнопке справа от поля.
- **PDF:** простая разметка с данными БСО и реквизитами исполнителя, встроенный шрифт для корректной кириллицы, кнопки «Скачать PDF» в реестре.
- **Реестр:** сохраняется в `localStorage`, отображает номер, дату/время, клиента, услугу, сумму, способ оплаты и даёт повторное скачивание PDF.
- **Данные продавца:** вынесены в `vendor.json` и подгружаются при инициализации. Для изменения реквизитов отредактируйте файл или замените его содержимое.

## Ограничения MVP
- Один пользователь, без аутентификации.
- Минималистичный дизайн: белый фон, чёрный текст, без адаптивной красоты.
- Данные хранятся только локально в браузере; при очистке локального хранилища история пропадает.
