# check_test

## MVP: функциональные требования
- Один пользователь, вход опционален.
- Форма «Создать БСО» с автоматической датой (сегодня) и ручным вводом времени.
- Поля: номер документа (авто), дата (авто), время, клиент (ФИО или «Физлицо»), услуга, сумма, способ оплаты (нал/безнал/СБП), комментарий (необязательно), реквизиты ИП (фиксированы или задаются один раз в настройках).
- Автоматическая нумерация БСО.
- Генерация PDF по фиксированному шаблону и кнопка «Скачать PDF».
- Реестр выданных БСО (список истории).

## Навигация
1. Главная: кнопка «Создать БСО» и таблица последних БСО.
2. Экран скачивания PDF после сохранения документа.
3. История: список всех созданных БСО.

## UX формы
Форма в одну колонку (одинакова на мобилке и десктопе):
- Клиент
- Услуга
- Сумма
- Время оплаты
- Способ оплаты (выпадающий список)
- Комментарий
- Кнопка «Создать PDF»

## Нумерация
MVP использует формулу `YYYYMMDD-XXX`, где `XXX` — последовательный номер в пределах дня с ведущими нулями (001, 002, …). Новая дата сбрасывает счётчик к 001.

## Минимальный дизайн
- Белый фон, чёрный текст.
- Без логотипов и декоративных цветов.
- Фиксированный неадаптивный layout: главное — проверка идеи.

## Дополнительные замечания для реализации
- Дата подставляется автоматически; время вводится вручную с валидацией формата.
- Реквизиты ИП вставляются в PDF из заранее сохранённого блока данных.
- PDF формируется по простому статичному шаблону: таблица полей БСО + реквизиты исполнителя.
- Реестр должен показывать номер, дату/время, клиента, услугу, сумму, способ оплаты и ссылку на скачивание PDF.
